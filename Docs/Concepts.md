# Course Concepts

| Concept | Purpose | Used In |
|---------|---------|---------|
| [Game Constants](Concepts/Game_Constants.md) | Centralize configuration values and eliminate magic numbers | All systems for consistent values |
| [Object Pooling](Concepts/Object_Pooling.md) | Optimize performance by reusing objects instead of instantiating/destroying | Projectiles, enemies, visual effects |
| [Observer Pattern](Concepts/Observer_Pattern.md) | Decouple systems through event-driven communication | Game events, UI updates, audio triggers |
| [Scriptable Objects](Concepts/Scriptable_Objects.md) | Create data-driven design for game configuration | Tower/Enemy data, wave configurations, game settings |
| [Update Manager](Concepts/Update_Manager.md) | Control update frequency and improve performance | Game loop systems, AI behaviors, visual updates |
| [Tweening](Concepts/Tweening.md) | Create smooth animations and transitions | UI animations, visual feedback, camera movements |
| [Abstraction](Concepts/Abstraction.md) | Define shared behavior through abstract base classes | Base tower/enemy classes, system foundations |
| [Interfaces](Concepts/Interfaces.md) | Define behavior contracts without implementation | Targeting systems, damage systems, game services |
